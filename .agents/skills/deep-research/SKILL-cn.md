# 深度研究

使用 firecrawl 和 exa MCP 工具从多个网页来源生成完整的、带引用的研究报告。

## 激活时机

- 用户要求对任意主题进行深度研究
- 竞争分析、技术评估或市场规模测算
- 对公司、投资者或技术的尽职调查
- 任何需要综合多个来源信息的问题
- 用户提及「research」、「deep dive」、「investigate」或「what's the current state of」

## MCP 要求

至少需要以下其中之一：
- **firecrawl** — `firecrawl_search`、`firecrawl_scrape`、`firecrawl_crawl`
- **exa** — `web_search_exa`、`web_search_advanced_exa`、`crawling_exa`

两者搭配使用可获得最佳覆盖范围。请在 `~/.claude.json` 或 `~/.codex/config.toml` 中配置。

## Workflow

### 步骤1：明确目标

提出1-2个快速澄清问题：
- 「你的目标是什么——学习、做决策，还是撰写内容？」
- 「你有任何特定的角度或深度要求吗？」

如果用户说「直接研究就行」——使用合理默认值跳过此步骤。

### 步骤2：规划研究

将主题拆分为3-5个研究子问题。示例：
- 主题：「AI 对医疗保健的影响」
  - 目前医疗保健领域的主要 AI 应用有哪些？
  - 已经衡量到的临床结果有哪些？
  - 监管方面的挑战有哪些？
  - 哪些公司在这个领域处于领先地位？
  - 市场规模和增长轨迹是怎样的？

### 步骤3：执行多源搜索

对每个子问题，使用可用的 MCP 工具进行搜索：

**使用 firecrawl：**
```
firecrawl_search(query: "<sub-question keywords>", limit: 8)
```

**使用 exa：**
```
web_search_exa(query: "<sub-question keywords>", numResults: 8)
web_search_advanced_exa(query: "<keywords>", numResults: 5, startPublishedDate: "2025-01-01")
```

**搜索策略：**
- 每个子问题使用2-3种不同的关键词变体
- 混合通用查询和聚焦新闻的查询
- 目标是总共获得15-30个独立来源
- 优先级：学术、官方、知名新闻 > 博客 > 论坛

### 步骤4：深度阅读核心来源

对最有价值的URL，获取完整内容：

**使用 firecrawl：**
```
firecrawl_scrape(url: "<url>")
```

**使用 exa：**
```
crawling_exa(url: "<url>", tokensNum: 5000)
```

完整阅读3-5个核心来源以获得深度信息。不要仅依赖搜索摘要。

### 步骤5：综合并撰写报告

报告结构：

```markdown
# [主题]：研究报告
*生成时间：[日期] | 来源数量：[N] | 可信度：[高/中/低]*

## 执行摘要
[3-5句话概述核心发现]

## 1. [第一个核心主题]
[带行内引用的发现]
- 核心要点 ([来源名称](url))
- 支持数据 ([来源名称](url))

## 2. [第二个核心主题]
...

## 3. [第三个核心主题]
...

## 关键要点
- [可落地的洞察1]
- [可落地的洞察2]
- [可落地的洞察3]

## 来源
1. [标题](url) — [一行总结]
2. ...

## 方法论
在网页和新闻中搜索了[N]个查询。分析了[M]个来源。
调研的子问题：[列表]
```

### 步骤6：交付

- **短主题**：在聊天中发布完整报告
- **长报告**：发布执行摘要 + 关键要点，将完整报告保存到文件

## 子代理并行研究

对于宽泛的主题，使用 Claude Code 的 Task 工具实现并行处理：

```
并行启动3个研究agent：
1. Agent 1：研究子问题1-2
2. Agent 2：研究子问题3-4
3. Agent 3：研究子问题5 + 交叉主题
```

每个agent进行搜索、阅读来源并返回发现。主会话将内容综合为最终报告。

## 质量规则

1. **每一项主张都需要来源**。没有无来源的断言。
2. **交叉验证**。如果仅有一个来源提及相关内容，将其标记为未验证。
3. **时效性很重要**。优先选择过去12个月内的来源。
4. **承认信息缺口**。如果你找不到某个子问题的优质信息，明确说明。
5. **禁止幻觉**。如果你不知道，就说「未找到足够数据」。
6. **区分事实与推断**。清晰标注估算、预测和观点内容。

## 示例

```
"研究核聚变能源的当前发展状态"
"2026年Rust与Go用于后端服务的深度对比"
"研究SaaS业务自力更生增长的最佳策略"
"当前美国住房市场的情况如何？"
"调研AI代码编辑器的竞争格局"
```

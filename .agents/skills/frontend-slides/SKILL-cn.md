# 前端幻灯片

创建零依赖、富含动画、完全在浏览器中运行的HTML演示文稿。

灵感来源于[zarazhangrui](https://github.com/zarazhangrui)作品中展示的视觉探索方法。

## 激活场景

- 创建演讲幻灯片、产品推介 deck、工作坊幻灯片或内部演示文稿
- 将`.ppt`或`.pptx`幻灯片转换为HTML演示文稿
- 优化现有HTML演示文稿的布局、动效或排版
- 与尚不明确设计偏好的用户一起探索演示文稿风格

## 硬性要求

1. **零依赖**：默认输出单个独立HTML文件，包含内联CSS和JS。
2. **视口适配为强制要求**：每张幻灯片必须完全适配单个视口，无内部滚动。
3. **展示而非告知**：使用视觉预览代替抽象的风格问卷。
4. **独特设计**：避免通用的紫色渐变、白色背景搭配Inter字体的模板化幻灯片。
5. **生产级质量**：保持代码有注释、可访问、响应式且高性能。

生成前请阅读`STYLE_PRESETS.md`了解视口安全的CSS基础、内容密度限制、预设目录和CSS注意事项。

## 工作流程

### 1. 确定模式

选择以下路径之一：
- **新建演示文稿**：用户有主题、笔记或完整草稿
- **PPT转换**：用户有`.ppt`或`.pptx`文件
- **优化增强**：用户已有HTML幻灯片，需要改进

### 2. 了解内容

仅询问必要的最少信息：
- 用途：产品推介、教学、会议演讲、内部更新
- 长度：短（5-10页）、中（10-20页）、长（20页以上）
- 内容状态：完稿、粗略笔记、仅主题

如果用户已有内容，请要求他们在样式设计前粘贴内容。

### 3. 探索风格

默认采用视觉探索方式。

如果用户已经知道想要的预设，跳过预览直接使用。

否则：
1. 询问幻灯片需要营造的氛围：印象深刻、充满活力、专注、启发灵感。
2. 在`.ecc-design/slide-previews/`目录下生成**3个单页预览文件**。
3. 每个预览必须是独立的，清晰展示排版/颜色/动效，幻灯片内容控制在约100行以内。
4. 询问用户要保留哪个预览，或想要混合哪些元素。

将氛围映射为风格时请使用`STYLE_PRESETS.md`中的预设指南。

### 4. 构建演示文稿

输出以下文件之一：
- `presentation.html`
- `[presentation-name].html`

仅当幻灯片包含提取的或用户提供的图片时，使用`assets/`文件夹。

必填结构：
- 语义化的幻灯片section
- 来自`STYLE_PRESETS.md`的视口安全CSS基础
- 用于主题值的CSS自定义属性
- 支持键盘、滚轮和触摸导航的演示文稿控制器类
- 用于显示动画的Intersection Observer
- 减少动效（reduced-motion）支持

### 5. 强制视口适配

将此作为硬性门槛。

规则：
- 每个`.slide`必须使用`height: 100vh; height: 100dvh; overflow: hidden;`
- 所有字体和间距必须使用`clamp()`进行缩放
- 内容放不下时，拆分为多张幻灯片
- 永远不要通过将文字缩小到可读尺寸以下来解决溢出问题
- 永远不要允许幻灯片内部出现滚动条

使用`STYLE_PRESETS.md`中的密度限制和强制CSS块。

### 6. 验证

在以下尺寸下检查完成的幻灯片：
- 1920x1080
- 1280x720
- 768x1024
- 375x667
- 667x375

如果有浏览器自动化可用，用它验证没有幻灯片溢出，且键盘导航正常工作。

### 7. 交付

交付时：
- 删除临时预览文件，除非用户想要保留
- 合适的情况下使用对应平台的打开命令打开幻灯片
- 总结文件路径、使用的预设、幻灯片数量和简易的主题自定义点

为当前操作系统使用正确的打开命令：
- macOS：`open file.html`
- Linux：`xdg-open file.html`
- Windows：`start "" file.html`

## PPT/PPTX转换

PowerPoint转换步骤：
1. 优先使用`python3`和`python-pptx`提取文字、图片和笔记。
2. 如果`python-pptx`不可用，询问用户是安装它还是回退到手动/基于导出的工作流。
3. 保留幻灯片顺序、演讲者备注和提取的资源。
4. 提取完成后，运行与新建演示文稿相同的风格选择工作流。

保持转换跨平台。如果Python可以完成工作，不要依赖仅macOS可用的工具。

## 实现要求

### HTML/CSS

- 使用内联CSS和JS，除非用户明确想要多文件项目。
- 字体可以来自Google Fonts或Fontshare。
- 优先使用氛围感背景、强排版层级和清晰的视觉方向。
- 使用抽象形状、渐变、网格、噪点和几何图形，而非插画。

### JavaScript

包含：
- 键盘导航
- 触摸/滑动导航
- 鼠标滚轮导航
- 进度指示器或幻灯片索引
- 进入时显示的动画触发器

### 可访问性

- 使用语义化结构（`main`、`section`、`nav`）
- 保持对比度可读
- 支持仅键盘导航
- 尊重`prefers-reduced-motion`设置

## 内容密度限制

除非用户明确要求更密集的幻灯片且不影响可读性，否则使用以下最大值：

| 幻灯片类型 | 限制 |
|------------|-------|
| 标题页 | 1个标题 + 1个副标题 + 可选标语 |
| 内容页 | 1个标题 + 4-6个项目符号或2个短段落 |
| 功能网格 | 最多6张卡片 |
| 代码页 | 最多8-10行 |
| 引语页 | 1条引语 + 署名 |
| 图片页 | 1张受视口限制的图片 |

## 反模式

- 没有视觉辨识度的通用创业公司渐变
- 使用系统字体的幻灯片，除非是有意的编辑风格
- 过长的项目符号列表
- 需要滚动的代码块
- 在短屏幕上会破损的固定高度内容框
- 无效的负CSS函数，如`-clamp(...)`

## 相关ECC技能

- `frontend-patterns`：用于幻灯片周边的组件和交互模式
- `liquid-glass-design`：当演示文稿有意借鉴Apple玻璃美学时使用
- `e2e-testing`：如果需要对最终幻灯片进行自动化浏览器验证时使用

## 交付检查清单

- 演示文稿可在浏览器中从本地文件运行
- 每张幻灯片都适配视口，无需滚动
- 风格独特且符合预期
- 动画有意义，不杂乱
- 尊重减少动效设置
- 交付时说明文件路径和自定义点

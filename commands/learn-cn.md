# /learn - 提取可复用模式

分析当前会话并提取所有值得保存为技能的模式。

## 触发条件

当你解决了一个非平凡问题后，可以在会话中的任何时候运行 `/learn`。

## 提取内容

查找以下内容：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这对于类似错误是否可复用？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方案**
   - 库的特殊行为
   - API 限制
   - 版本特定的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 位置创建技能文件：

```markdown
# [描述性的模式名称]

**提取时间:** [日期]
**上下文:** [适用场景的简要描述]

## 问题
[该模式解决的问题 - 请具体说明]

## 解决方案
[模式/技术/变通方案]

## 示例
[适用的话提供代码示例]

## 使用时机
[触发条件 - 什么情况下应该激活该技能]
```

## 流程

1. 回顾会话寻找可提取的模式
2. 识别最有价值/可复用的洞察
3. 起草技能文件
4. 保存前询问用户确认
5. 保存到 `~/.claude/skills/learned/` 目录

## 注意事项

- 不要提取微不足道的修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定 API 中断等）
- 关注能在未来会话中节省时间的模式
- 保持技能聚焦 - 每个技能只对应一个模式

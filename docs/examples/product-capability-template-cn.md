# 产品能力模板

当产品意图已存在但实现约束仍不明确时使用此模板。

目的是创建一个持久的能力契约，而不是另一份模糊的规划文档。

## 能力

- **能力名称：**
- **来源：** PRD / issue / discussion / roadmap / founder note
- **主要参与者：**
- **发布后成果：**
- **成功信号：**

## 产品意图

用一个简短段落描述用户可见的承诺。

## 约束

列出实现开始前必须满足的规则：

- 业务规则
- 范围边界
- 不变量
- 发布约束
- 迁移约束
- 向后兼容性约束
- 计费 / 认证 / 合规约束

## 参与者与交互面

- 参与者
- UI 交互面
- API 交互面
- 自动化 / 操作员交互面
- 报告 / 仪表板交互面

## 状态与转换

用明确的状态和允许的转换来描述生命周期。

示例：

- `draft -> active -> paused -> completed`
- `pending -> approved -> provisioned -> revoked`

## 接口契约

- 输入
- 输出
- 必需的副作用
- 失败状态
- 重试 / 恢复
- 幂等性期望

## 数据影响

- 真实数据源 (source of truth)
- 新实体或字段
- 所有权边界
- 保留 / 删除期望

## 安全与策略

- 信任边界
- 权限要求
- 滥用路径
- 策略 / 治理要求

## 非目标

列出此能力明确不负责的内容。

## 待解决问题

记录阻碍实现的未解决决策。

## 交接

- **是否准备好实现？**
- **是否需要架构评审？**
- **是否需要产品澄清？**
- **下一个 ECC 流程：** `project-flow-ops` / `tdd-workflow` / `verification-loop` / 其他

# /learn - 可重用模式提取

分析当前会话，提取值得作为 skill 保存的模式。

## Trigger

在解决非平凡问题后，随时在会话中运行 `/learn`。

## 要提取的内容

查找以下内容：

1. **错误解决模式**
   - 发生了什么错误
   - 根本原因是什么
   - 什么修复了它
   - 是否可重用于类似错误

2. **调试技术**
   - 非平凡的调试步骤
   - 效果良好的工具组合
   - 诊断模式

3. **变通方案**
   - 库的特性
   - API 限制
   - 版本特定修复

4. **项目特定模式**
   - 发现的代码库约定
   - 已做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[模式名].md` 创建 skill 文件：

```markdown
# [描述性模式名称]

**提取日期:** [日期]
**上下文:** [何时适用的简要说明]

## 问题
[要解决的问题 - 具体说明]

## 解决方案
[模式/技术/变通方案]

## 示例
[代码示例（如适用）]

## 使用时机
[触发条件 - 应启用此 skill 的情况]
```

## 流程

1. 审查会话中可提取的模式
2. 确定最有价值/可重用的洞察
3. 起草 skill 文件
4. 保存前寻求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不提取琐碎修复（打字错误、简单语法错误）
- 不提取一次性问题（如特定 API 故障）
- 专注于可为未来会话节省时间的模式
- 保持 skills 聚焦 - 一个 skill 对应一个模式

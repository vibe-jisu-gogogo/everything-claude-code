# Refactor Clean

通过测试验证安全地识别和删除死代码:

1. 运行死代码分析工具:
   - knip: 检测未使用的导出和文件
   - depcheck: 检测未使用的依赖
   - ts-prune: 检测未使用的TypeScript导出

2. 在 .reports/dead-code-analysis.md 生成综合报告

3. 按重要性分类发现:
   - SAFE: 测试文件、未使用的工具
   - CAUTION: API路由、组件
   - DANGER: 配置文件、主入口点

4. 仅建议安全删除

5. 每次删除前:
   - 运行完整测试套件
   - 确认测试通过
   - 应用变更
   - 重新运行测试
   - 如果测试失败则回滚

6. 显示清理项目的摘要

切勿在未先运行测试的情况下删除代码!

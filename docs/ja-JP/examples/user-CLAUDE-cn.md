# 用户级 CLAUDE.md 示例

这是用户级 CLAUDE.md 文件的示例。请放置在 `~/.claude/CLAUDE.md`。

用户级设置全局应用于所有项目。用于以下用途:
- 个人编码设置
- 希望始终应用的通用规则
- 模块化规则的链接

---

## 核心哲学

你是Claude Code。我使用专注于复杂任务的代理和技能。

**主要原则:**
1. **代理优先**: 将复杂工作委托给专业代理
2. **并行执行**: 尽可能对多个代理使用Task工具
3. **先计划再执行**: 对复杂操作使用Plan Mode
4. **测试驱动**: 在实现前编写测试
5. **安全优先**: 不妥协于安全

## 个人设置

### 隐私
- 始终编辑日志; 不粘贴机密信息(API密钥/令牌/密码/JWT)
- 分享前审查输出 - 删除所有敏感数据

### 代码风格
- 不在代码、注释、文档中使用emoji
- 优先不变性 - 永远不修改对象或数组
- 多个小文件胜于少数大文件
- 通常200-400行，每个文件最大800行

### Git
- Conventional Commits: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- 提交前始终在本地测试
- 小而聚焦的提交

### 测试
- TDD: 先写测试
- 最低80%覆盖率
- 关键流程需要单元 + 集成 + E2E测试

## 成功指标

满足以下条件即为成功:
- 所有测试通过 (80%以上覆盖率)
- 无安全漏洞
- 代码易读且可维护
- 满足用户需求

# 通用模式

## 骨架项目

实现新功能时：
1. 搜索经过实战测试的骨架项目
2. 使用并行 agent 评估选项：
   - 安全性评估
   - 扩展性分析
   - 相关性评分
   - 实现计划
3. 基于最佳选项进行克隆
4. 在经过验证的结构内迭代

## 设计模式

### Repository 模式

在统一接口后封装数据访问：
- 定义标准操作：findAll、findById、create、update、delete
- 具体实现处理存储细节（数据库、API、文件等）
- 业务逻辑依赖抽象接口，而非存储机制
- 支持轻松交换数据源，简化通过 mock 进行的测试

### API 响应格式

对所有 API 响应使用一致的封装：
- 包含成功/状态指示器
- 包含数据 payload（出错时为 null）
- 包含错误消息字段（成功时为 null）
- 分页响应包含元数据（total、page、limit）

# 性能优化

## 模型选择策略

**Haiku 4.5**（Sonnet 功能的 90%，成本 1/3）:
- 频繁调用的轻量级 agent
- 结对编程和代码生成
- 多 agent 系统的 worker agent

**Sonnet 4.5**（最佳编码模型）:
- 主要开发工作
- 多 agent 工作流的编排
- 复杂编码任务

**Opus 4.5**（最深层次推理）:
- 复杂架构决策
- 最大推理需求
- 调查与分析任务

## 上下文窗口管理

在以下情况避免使用上下文窗口的最后 20%:
- 大规模重构
- 跨多个文件的功能实现
- 复杂交互的调试

上下文敏感度低的任务:
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单 bug 修复

## 扩展思考 + 计划模式

扩展思考默认启用，为内部推理预留最多 31,999 个 token。

扩展思考控制:
- **切换**: Option+T（macOS）/ Alt+T（Windows/Linux）
- **设置**: 在 `~/.claude/settings.json` 中设置 `alwaysThinkingEnabled`
- **预算上限**: `export MAX_THINKING_TOKENS=10000`
- **详细模式**: Ctrl+O 显示思考输出

对于需要深度推理的复杂任务:
1. 确认扩展思考已启用（默认启用）
2. 启用**计划模式**以获得结构化方法
3. 使用多轮批判进行彻底分析
4. 使用角色分工的子 agent 获得多样化视角

## 构建故障排除

构建失败时:
1. 使用 **build-error-resolver** agent
2. 分析错误消息
3. 逐步修复
4. 每次修复后验证

# 安全指南

## 必需安全检查

所有提交前:
- [ ] 无硬编码的密钥（API keys、passwords、tokens）
- [ ] 所有用户输入已验证
- [ ] SQL 注入预防（参数化查询）
- [ ] XSS 预防（经过净化的 HTML）
- [ ] CSRF 保护已启用
- [ ] 认证/授权已验证
- [ ] 所有端点都有速率限制
- [ ] 错误消息不泄露敏感数据

## 密钥管理

- 不要在源代码中硬编码密钥
- 始终使用环境变量或密钥管理器
- 验证启动时所需的密钥是否存在
- 轮换可能已暴露的密钥

## 安全响应协议

如果发现安全问题:
1. 立即停止
2. 使用 **security-reviewer** agent
3. 在继续前修复 CRITICAL 问题
4. 轮换已暴露的密钥
5. 检查整个代码库是否存在类似问题

# 测试要求

## 最低测试覆盖率: 80%

测试类型（全部必填）:
1. **Unit Test** - 单个函数、工具类、组件
2. **Integration Test** - API 端点、数据库操作
3. **E2E Test** - 关键用户流程（框架按语言选择）

## 测试驱动开发

必须工作流:
1. 先写测试（RED）
2. 运行测试 - 应该失败
3. 写最小实现（GREEN）
4. 运行测试 - 应该通过
5. 重构（IMPROVE）
6. 确认覆盖率（80%+）

## 测试失败排查

1. 使用 **tdd-guide** agent
2. 确认测试隔离
3. 验证 Mock 正确性
4. 修复实现，不修复测试（除非测试本身错误）

## Agent 支持

- **tdd-guide** - 新功能主动使用，强制测试优先

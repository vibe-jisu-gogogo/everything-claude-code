# /learn - 可重用模式提取

分析当前会话并提取值得保存为 skill 的模式。

## 触发条件

当在会话中解决了重要问题时，执行 `/learn`。

## 提取对象

寻找以下内容：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 是什么解决了问题？
   - 是否可以重用于类似的错误？

2. **调试技术**
   - 不直观的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **绕过方法**
   - 库的特殊情况
   - API 限制事项
   - 版本相关修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 生成 skill 文件：

```markdown
# [描述性模式名称]

**提取日期：** [日期]
**上下文：** [简要描述此模式适用的情况]

## 问题
[此模式解决的问题 - 具体描述]

## 解决方法
[模式/技术/绕过方法]

## 示例
[如适用，提供代码示例]

## 使用时机
[触发条件 - 此 skill 应被激活的情况]
```

## 流程

1. 审查会话中可提取的模式
2. 识别最有价值且可重用的洞察
3. 起草 skill 文件
4. 保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不要提取细微的修改（拼写错误、简单语法错误）
- 不要提取一次性问题（特定 API 故障等）
- 专注于能在未来会话中节省时间的模式
- 保持 skill 专注 - 每个 skill 一个模式

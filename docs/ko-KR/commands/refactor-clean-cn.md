# Refactor Clean

安全识别未使用的代码并在每个步骤执行测试验证以将其移除。

## 1阶段: 检测未使用的代码

根据项目类型运行分析工具:

| 工具 | 检测对象 | 命令 |
|------|----------|------|
| knip | 未使用的 exports、文件、依赖 | `npx knip` |
| depcheck | 未使用的 npm 依赖 | `npx depcheck` |
| ts-prune | 未使用的 TypeScript exports | `npx ts-prune` |
| vulture | 未使用的 Python 代码 | `vulture src/` |
| deadcode | 未使用的 Go 代码 | `deadcode ./...` |
| cargo-udeps | 未使用的 Rust 依赖 | `cargo +nightly udeps` |

如果没有可用的工具，请使用 Grep 查找没有被 import 的 export:
```
# 找到 export 后，检查是否在其他地方被 import
```

## 2阶段: 结果分类

按安全等级对结果进行分类:

| 等级 | 示例 | 措施 |
|------|------|------|
| **安全** | 未使用的实用程序、测试辅助工具、内部函数 | 放心删除 |
| **注意** | 组件、API 路由、中间件 | 确认是否有动态 import 或外部使用者 |
| **危险** | 配置文件、入口点、类型定义 | 处理前需要调查 |

## 3阶段: 安全删除循环

对每个安全项:

1. **运行完整测试套件** --- 建立基准线 (全部通过)
2. **删除未使用的代码** --- 使用 Edit 工具精确移除
3. **重新运行测试套件** --- 确认没有破坏任何功能
4. **测试失败时** --- 立即用 `git checkout -- <file>` 回滚并跳过该项目
5. **测试通过时** --- 移至下一项

## 4阶段: 处理注意项

在删除注意项之前:
- 搜索动态 import: `import()`, `require()`, `__import__`
- 搜索字符串引用: 路由名称、配置文件中的组件名称
- 确认是否在公开包 API 中被 export
- 确认没有外部使用者 (如果已发布，检查依赖包)

## 5阶段: 重复项整合

删除未使用的代码后，查找以下内容:
- 几乎重复的函数 (80% 以上相似度) --- 合并为一个
- 重复的类型定义 --- 整合
- 不增加价值的包装函数 --- 内联处理
- 无目的的 re-export --- 移除间接引用

## 6阶段: 总结

报告结果:

```
Dead Code Cleanup
──────────────────────────────
删除:     未使用函数 12个
           未使用文件 3个
           未使用依赖 5个
跳过:     项目 2个 (测试失败)
减少:     约 450行 已移除
──────────────────────────────
所有测试通过 PASS:
```

## 规则

- **绝对不要在运行测试前删除**
- **一次只删除一个** --- 原子化变更使回滚更容易
- **不确定就跳过** --- 保持未使用的代码比破坏生产环境更好
- **清理时不要重构** --- 关注点分离 (先清理，后重构)

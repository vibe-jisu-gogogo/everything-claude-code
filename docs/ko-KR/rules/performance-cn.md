# 性能优化

## 模型选择策略

**Haiku 4.5** (Sonnet 能力的 90%，节省 3 倍成本):
- 频繁调用的轻量级代理
- 结对编程和代码生成
- 多代理系统中的工作代理

**Sonnet 4.6** (最佳编码模型):
- 主要开发任务
- 多代理工作流编排
- 复杂编码任务

**Opus 4.5** (最深推理能力):
- 复杂架构决策
- 最大推理需求
- 研究和分析任务

## 上下文窗口管理

在上下文窗口的最后 20% 应避免：
- 大规模重构
- 跨多个文件的功能实现
- 复杂交互调试

上下文敏感度低的任务：
- 单文件修改
- 独立工具创建
- 文档更新
- 简单错误修复

## 扩展思维 + 计划模式

扩展思维默认启用，为内部推理预留最多 31,999 tokens。

扩展思维控制方法：
- **切换**: Option+T (macOS) / Alt+T (Windows/Linux)
- **设置**: 在 `~/.claude/settings.json` 中配置 `alwaysThinkingEnabled`
- **预算限制**: `export MAX_THINKING_TOKENS=10000`
- **详细模式**: Ctrl+O 查看思维输出

需要深度推理的复杂任务：
1. 确认扩展思维已启用（默认启用）
2. 启用**计划模式**进行结构化处理
3. 执行多轮批判以进行彻底分析
4. 使用角色分离子代理以获得多样化视角

## 构建问题解决

构建失败时：
1. 使用 **build-error-resolver** 代理
2. 分析错误消息
3. 逐步修复
4. 每次修复后验证

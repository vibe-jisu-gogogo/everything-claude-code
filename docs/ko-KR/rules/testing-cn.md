# 测试要求

## 最低测试覆盖率: 80%

测试类型 (全部必需):
1. **单元测试** - 个别函数、utility、component
2. **集成测试** - API endpoint、数据库操作
3. **E2E 测试** - 核心用户流程 (按语言选择 framework)

## 测试驱动开发

必需工作流程:
1. 先编写测试 (RED)
2. 执行测试 - 应该失败
3. 编写最少量的实现 (GREEN)
4. 执行测试 - 应该通过
5. 重构 (IMPROVE)
6. 确认覆盖率 (80% 以上)

## 测试失败问题解决

1. **tdd-guide** agent 使用
2. 确认测试隔离
3. 验证 mocking 是否正确
4. 修改实现而非测试 (除非测试本身错误)

## Agent 支持

- **tdd-guide** - 积极用于新功能，强制先编写测试

# /learn - 提取可复用模式

分析当前会话并提取值得保存为 skills 的模式。

## 触发条件

当你解决了一个非平凡问题后，在会话的任何时刻运行 `/learn`。

## 提取内容

寻找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这对类似错误是否可复用？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方案**
   - 库的特殊行为 (quirks)
   - API 限制
   - 特定版本的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建 skill 文件：

```markdown
# [描述性模式名称]

**Extracted:** [日期]
**Context:** [适用场景的简要描述]

## Problem
[解决的问题 - 请具体]

## Solution
[模式/技术/变通方案]

## Example
[适用的代码示例]

## When to Use
[触发条件 - 什么应该激活该 skill]
```

## 流程

1. 回顾会话以识别可提取的模式
2. 识别最有价值/可复用的见解
3. 草拟 skill 文件
4. 保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不要提取琐碎的修复（拼写错误、简单的语法错误）
- 不要提取一次性问题（特定的 API 不可用等）
- 专注于能在未来会话节省时间的模式
- 保持 skills 专注 - 每个 skill 一个模式

# 通用模式

## Skeleton 项目

实现新功能时：
1. 寻找经过良好测试的 skeleton 项目
2. 使用并行代理评估选项：
   - 安全性评估
   - 可扩展性分析
   - 相关性评分
   - 实施计划
3. 克隆最佳选项作为基础
4. 在经过验证的架构内迭代

## 设计模式

### Repository 模式

将数据访问封装在一致的接口后：
- 定义标准操作：findAll、findById、create、update、delete
- 具体实现处理存储细节（数据库、API、文件等）
- 业务逻辑依赖抽象接口，而非存储机制
- 支持轻松切换数据源并简化 mock 测试

### API 响应格式

为所有 API 响应使用一致的信封：
- 包含成功/状态指示器
- 包含数据 payload（出错时可为空）
- 包含错误消息字段（成功时可为空）
- 包含分页响应的元数据（total、page、limit）

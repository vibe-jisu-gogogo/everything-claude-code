# 测试要求

## 最小测试覆盖率：80%

测试类型（全部为必需）：
1. **单元测试** - 单个函数、工具、组件
2. **集成测试** - API endpoints、数据库操作
3. **E2E测试** - 关键用户流程（框架按语言选择）

## 测试驱动开发 (TDD)

必须遵循的工作流程：
1. 先写测试 (RED)
2. 运行测试 - 必须失败
3. 编写最小实现 (GREEN)
4. 运行测试 - 必须通过
5. 重构 (REFACTOR)
6. 检查覆盖率 (80%+)

## 测试失败解决

1. 使用 **tdd-guide** agent
2. 检查测试隔离
3. 检查 mocks 是否正确
4. 修复实现，不要修复测试（除非测试本身是错误的）

## Agent 支持

- **tdd-guide** - 新功能时主动使用，强制执行先写测试
